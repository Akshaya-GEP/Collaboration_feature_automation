Agentic nodes testing (Playwright + Cucumber)

Goal

Add test coverage for all specified nodes and flows by writing feature files and step definitions in the existing Playwright + Cucumber framework. Node behavior is validated only through UI/API flows (create workflow, configure nodes, run, assert). No unit tests or mocks inside services/libs/agentic-engine.

Process (order of work)

Step 1 – Scenario inventory
Write one document that lists every scenario to implement (e.g. in playwright_automation/bdd/features/NODES_TEST_PLAN.md). For each node and integration flow, write one line per scenario, e.g. “Create workflow with LLM node, run, validate response in chat.” No code yet.

Step 2 – Feature files
Create or update .feature files under playwright_automation/bdd/features/ so each scenario from the inventory becomes a Gherkin Scenario or Scenario Outline. Use high-level steps: open app, login, add node(s), configure, link, save, run, assert result. Reuse existing steps (e.g. from workflow.steps.ts, AgentCreation.steps.ts, LLMValidation.steps.ts) where possible. Add tags (e.g. @nodes_llm, @handoff) for filtering.

Step 3 – Test data
Add JSON (or other) fixtures in playwright_automation/bdd/data/ for workflows that need fixed structure (e.g. Rule with two branches, Variable + Agent, Handoff chain). Reference these in feature steps (e.g. “paste workflow from …”) so scenarios are repeatable.

Step 4 – Step definitions
Implement only the steps that don’t exist yet: new step definitions in bdd/step-definitions/ and, if needed, new or updated page objects under pages/. Keep steps at UI/flow level (e.g. “add Rule node,” “assert run completed”), not node-internals.

Step 5 – Run and tag
Run BDD via existing commands (e.g. npm run bdd:unified). Use tags to run by area (e.g. only @nodes_rule or @handoff_integration).
